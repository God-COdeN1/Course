WEB фреймворк Flask

Flask - це мікрофреймворк для створення веб-додатків мовою програмування Python. Він надає базові інструменти та структуру для створення веб-додатків, але при цьому залишає багато рішень і архітектурних деталей на розсуд розробника. Flask був розроблений з метою бути легким, гнучким і простим у використанні.
Деякі ключові особливості Flask:

  • Мінімалізм: Flask надає мінімальний набір інструментів для створення веб-додатків. Це дозволяє розробникам вибирати та інтегрувати необхідні компоненти за їхнім бажанням.
  • Маршрутизація: Flask дозволяє визначити URL-адреси (маршрути), які додаток буде обробляти. Це робиться з використанням декораторів Python.
  • Шаблони: Flask підтримує використання шаблонів (наприклад, Jinja2), що спрощує відображення динамічних даних на веб-сторінках.
  • HTTP-методи: Додатки на Flask можуть обробляти різні HTTP-методи, такі як GET, POST, PUT, DELETE і інші.
  • Розширюваність: Існує багато розширень (extensions) для Flask, які додають додаткові функціональні можливості, такі як робота з базами даних, аутентифікація, обробка форм та інше.
  • Необхідність самостійних рішень: Flask залишає багато архітектурних рішень на розсуд розробника. Це надає велику свободу, але також вимагає від розробників приймати рішення щодо організації коду і структури додатку.

  • Спочатку ми імпортували клас Flask. Примірник цього класу і буде вашим додатком WSGI.
  • Далі ми створюємо екземпляр цього класу. Перший аргумент – це ім'я модуля або пакета програми.   • Далі, ми використовуємо декоратор route(), щоб сказати Flask, який URL повинен запускати нашу функцію.

Механізм шаблонів і мову шаблонів Jinja2

Jinja2 - це популярний і потужний механізм шаблонів для Python, який використовується веб-фреймворками, такими як Flask і Django, а також в інших проектах Python для генерації тексту на основі шаблонів. Він дозволяє розробникам вставляти змінні та логіку в шаблони для створення динамічного вмісту, який може бути відображений на веб-сторінках або в інших текстових документах.
Основні концепції Jinja2:

  • Змінні: Змінні вставляються в шаблони за допомогою подвійних фігурних дужок {{ }}. Наприклад, {{ username }} вставляє значення змінної username в шаблон.
  • Вирази: Вирази дозволяють виконувати обчислення та операції в шаблоні за допомогою фігурних дужок з відсічкою {% %}. Наприклад, {% for item in items %} використовується для ітерації через список items.
  • Умовні оператори: Ви можете вставляти умовні оператори в шаблони за допомогою {% if ... %} та {% else %}. Наприклад:
  • Цикли: Jinja2 підтримує цикли для обробки списків або інших ітерабельних об'єктів. Наприклад:
  • Макрофункції: Ви можете оголошувати та викликати макрофункції в шаблоні, що дозволяє виокремити і перевикористовувати фрагменти коду.
  • Спадкування шаблонів: Ви можете створювати батьківські та дочірні шаблони для використання спадкування та відновлення частин шаблонів.
  • Фільтри та розширення: Jinja2 має багато вбудованих фільтрів та можливість розширення функціональності шаблонів за допомогою розширень.
  • Безпека: Jinja2 має вбудовану підтримку безпеки і автоматично екранує дані, що вставляються в шаблони, для запобігання атакам внедрення коду.

Загалом, Jinja2 дозволяє створювати динамічні веб-сторінки та інші тексові документи, вставляючи змінні, умови та логіку безпосередньо в шаблони. Він є важливою складовою багатьох веб-фреймворків та додатків, які використовують Python для розробки веб-додатків та генерації HTML-сторінок.

Сесії і форми

Сесії та форми - це важливі складові веб-розробки, які використовуються для збереження стану і обробки даних у веб-додатках. Вони є ключовими елементами для взаємодії з користувачами та збереження даних між HTTP-запитами.

Сесії:

  • Сесія - це механізм, який дозволяє зберігати дані на сервері між різними HTTP-запитами в рамках одного користувача. Вона дозволяє створювати та зберігати змінні, які будуть доступні під час всієї сесії користувача.
  • Зазвичай сесії використовуються для збереження інформації про стан користувача, таку як ідентифікатори сесій, дані про входження, корзини для покупок тощо.
  • Для зберігання сесій в Flask і багатьох інших веб-фреймворках використовуються кукі або серверна пам'ять. Ви можете зберігати у сесії будь-які дані, які потрібні для вашого додатка.
  • Щоб встановити сесію в Flask, вам спочатку потрібно ініціалізувати її, а потім зберігати та зчитувати дані з неї. Наприклад:

Форми:

  • Форма - це інтерактивний елемент веб-сторінки, який дозволяє користувачам вводити та надсилати дані на сервер для обробки.
  • Форми використовуються для збору різних видів інформації від користувачів, таких як реєстраційні дані, коментарі, пошукові запити тощо.
  • Для створення та обробки форм в веб-додатках Flask використовуються розширення, такі як Flask-WTF або вбудовані засоби бібліотеки request.

WEB-фреймворк Bottle

Bottle - це мінімалістичний мікрофреймворк для веб-розробки на мові програмування Python. Як і інші мікрофреймворки, Bottle надає базовий набір інструментів для створення веб-додатків, але при цьому має невеликий розмір та низький поріг входження. Основною метою Bottle є надання швидкого і простого рішення для створення веб-додатків невеликого розміру.
Особливості Bottle:

  • Мінімалізм: Bottle має дуже низький рівень абстракції, що дозволяє розробникам швидко створювати прості веб-додатки.
  • Один файл: Один додаток Bottle може бути представлений одним файлом. Це зручно для невеликих проектів.
  • Вбудований сервер: Bottle поставляється з вбудованим веб-сервером, що дозволяє запускати додаток без необхідності встановлення окремого веб-сервера.
  • Маршрутизація: Маршрутизація в Bottle відбувається за допомогою декораторів, які вказують, який обробник має бути викликаний для конкретного URL.
  • Шаблонізація: Bottle підтримує використання вбудованої шаблонної мови для відображення динамічних даних на сторінках.
  • Сесії: Bottle надає можливість роботи з сесіями для зберігання стану між різними HTTP-запитами.
  • Розширення: Хоча Bottle і має мінімалістичний характер, він має плагіни, які додають додаткові можливості, такі як робота з базами даних, аутентифікація тощо.
